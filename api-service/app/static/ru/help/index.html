Помощь: Использование API-сервиса.

1. Регистрация аккаунта.

  1. Перейдите на страницу регистрации, кликнув по ссылке "Создать аккаунт".
  2. Введите свою почту и пароль. Подтвердите пароль.
    Пароль должен содержать не менее 8 символов.
  3. Согласитесь с нашим Политикой конфиденциальности.
  4. Нажмите на кнопку "Регистрация" для создания вашего аккаунта.
  5. Вы будете уведомлены о создании вашего аккаунта на почте, которую вы указали.
  6. Вы можете теперь вернуться на страницу входа и войти в свой аккаунт.

2. Изменение вашего пароля

  1. Ввойдите в свой аккаунт.
  2. Измените пароль в странице пользователя.
  3. Введите ваш текущий пароль, новый пароль и подтвердите новый пароль.
  4. Нажмите на кнопку "Изменить пароль" для изменения вашего пароля.
  5. Вы будете уведомлены об изменении вашего пароля на вашей почте.

3. Удаление вашего аккаунта

  1. Ввойдите в своий аккаунт.
  2. Нажмите на кнопку "Удалить" для открытия формы удаления аккаунта.
  3. Введите ваш пароль, чтобы подтвердить ваше намерение удалить своий аккаунт.
  4. Подтвердите, что вы согласны с удалением своего аккаунта.
  5. Нажмите на кнопку "Удалить" для удаления вашего аккаунта.

Внимание: Удаление вашего аккаунта приведет к удалению всех ваших данных из системы.
Пожалуйста, прежде чем продолжить, убедитесь, что вы хотите удалить своий аккаунт.

4. Управление токенами

  1. Вы должны быть регистрированы, чтобы получить токен.
  2. Запросите публичный ключ из public key api endpoint.
  3. Зашифруйте ваш пароль и почту с помощью публичного ключа используя RSA алгоритм.
  4. Для web-js вы можете использовать готовый RSA файл и запросить HttpOnly токен из web-token api endpoint.
  5. Для python вы должны использовать PyCryptodome библиотеку и SHA256 алгоритм
    и запросить не HttpOnly токен из token api endpoint.
  6. Используйте POST метод для отправки зашифрованных пароля и почты в token api endpoint вашего выбора.
  7. Если вы храните токен в cookies используйте GET метод.
  8. Если вы включаете токен в тело запроса используйте POST метод. Назовите токен "token" в теле запроса.
  9. Для подробной информации ознакомьтесь с api endpoints документацией.
  10. Токен является действительныйм 3 часа.
